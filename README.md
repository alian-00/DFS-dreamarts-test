# 最長片道きっぷ探索 CLI

DreamArts のコーディングテスト向けに、Python 3.12 と標準ライブラリのみで実装した CLI ツールです。

標準入力から無向重み付きグラフを読み込み、同じ駅を 2 回通らない条件で到達可能な最長経路を探索し、経路上の駅 ID を 1 行ずつ標準出力へ出します。

## 要求要件

- Python 3.12 以上
- 外部ライブラリ不要

## 使用方法

```bash
python3 -m src.main < input.txt
```

サンプルデータですぐ試す場合:

```bash
python3 -m src.main < sample_input.txt
```

入力フォーマット:

```text
start_id, end_id, distance
```

- 駅 ID は正の整数
- 距離は 0 以上の数値
- 行頭行末の空白は許容
- 空行は無視

出力フォーマット:

```text
station_id
station_id
station_id
```

## 実行例

入力:

```text
1, 2, 8.54
2, 3, 3.11
3, 1, 2.19
3, 4, 4
4, 1, 1.4
```

出力:

```text
1
2
3
4
```

この例の期待結果は `[sample_output.txt](/Users/alian/Documents/dreamarts-test/sample_output.txt)` にも入れています。

## 設計メモ

- 入力処理、経路探索、CLI の入出力を分けて、役割が分かりやすい構成にしました。
- 路線情報は、各駅からどこへ移動できるかを扱いやすい形に整理しています。
- 経路探索では、同じ駅を2回通らないように訪問済みの駅を管理しながら、候補を順番に調べて最長の経路を選んでいます。
- 結果が毎回ぶれないように、探索する順番を固定しています。

## エラー処理

- 入力形式が正しくない場合は、エラーメッセージを表示して終了するようにしています。
- 駅IDや距離が不正な値だった場合は、探索処理に進まないようにしています。
- 入力が空の場合は、何も出力せず正常終了するようにしています。
